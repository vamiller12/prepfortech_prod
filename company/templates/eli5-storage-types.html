TL;DR - Object Storage vs File Storage vs Block Storage:

Each file storage system allows data to store with different metadata and accessibility options. Customizable metadata for an object is important in the data analytics realm. The attributes are important about an object may change depending on the object such as type of device and IP address. The accessibility options become important where there is a large amount of data or multiple programs trying to read or write from the database simultaneously. In a file storage all requests go down the same file pathway which can lead to requests having to wait for others to be completed. In object and block storage, the data can be stored across locations providing flexibility for the system on accomplishing requests.

Data Formats:

When deciding on while file storage system to employ, the data format is an important and potentially limiting factor. Data takes two high level formats: structured and unstructured.

  • Unstructured: Data that does not have a pre-defined data model. This is the common storage technique for big data since it is far more flexible.
  • Structured: Data that is stored in a schema that employs a rigid form. This is typically found in a relational database where data is split into tables and associated by keys.
