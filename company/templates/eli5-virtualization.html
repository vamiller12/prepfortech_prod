TL;DR - Virtualization

The process of abstracting the hardware from the software via a hypervisor in order to utilize the full capability of the underlying hardware and/or simplify maintenance and management. By decoupling the software from the hardware, the hypervisor allows the hardware device to manipulated for business purposes. Applications of virtualization options exist from the complexities of data center hardware and networks, to the simpler the process of virtualizing desktops.

Examples of Virtualization:
  • Storage Virtualization: Combining multiple physical storage devices to appear as a single, large storage source. This approach allows for flexibility in designing.
  • VDI (Virtual Desktop Infrastructure): Creating the appearance of multiple desktops by adding virtual machines to the actual desktop. This allows for easier maintenance and lower costs.
  • Virtual Private Network (VPN): A virtualization of the private network that allows users not in the private network range to connect to the network and anything (applications, data, etc.) secured on the network.
  • Virtual Network Function (VNFs): A networking architecture where the network services are virtualized in order to configure and share components as needed.
